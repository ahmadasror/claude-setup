# PRD — Orders Module

**Status**: Draft · **Generated**: 2026-04-20 by requirement-gatherer · **Version**: 0.1

## Executive Summary

Orders module menangani end-to-end siklus pesanan pelanggan: dari submit order, validasi stok & payment, dispatch ke fulfillment, sampai closure. Target: mendukung katalog ~50K SKU, traffic peak 200 order/menit, dan compliance dengan konsumen regulasi lokal.

**Business goals**
1. Reduce order abandonment rate dari 28% ke ≤ 15% dalam 2 kuartal
2. SLA confirmation ≤ 3 detik P95 untuk order eligible
3. Audit trail lengkap untuk setiap state transition (regulatory requirement)

## Personas

| Persona | Role | Primary goal |

| Customer | End user yang melakukan pembelian | Order cepat diconfirm, transparansi status |

## Flow Map

### F-01 — Customer submit order (happy path)

- **Trigger**: Customer klik "Place Order" di checkout
- **Actor**: Customer (authenticated)
- **Outcome**: Order tercipta di state `CONFIRMED`, stock reserved, payment authorized, fulfillment dispatched
- **Prerequisite**: Cart non-empty, payment method valid, shipping address valid
- **Business steps**:
  1. Validate cart snapshot (price & availability)
  2. Reserve inventory
  3. Authorize payment
  4. Create order record → state `PENDING`
  5. Transition → `CONFIRMED`
  6. Dispatch fulfillment + send confirmation notif
- **PRD workflows**: `order-submit`, `inventory-reserve`, `payment-authorize`, `fulfillment-dispatch`

### F-02 — Payment declined

- **Trigger**: Payment gateway returns decline
- **Actor**: Customer
- **Outcome**: Order tidak tercipta, cart dipertahankan, customer dapat notif decline reason
- **Prerequisite**: F-01 steps 1-2 berhasil, step 3 gagal
- **PRD workflows**: `order-submit`, `payment-authorize`

### F-03 — Customer cancel before dispatch

- **Trigger**: Customer klik "Cancel" saat order di state `CONFIRMED` tapi belum `DISPATCHED`
- **Actor**: Customer
- **Outcome**: Order → `CANCELLED`, stock released, payment voided/refunded
- **Prerequisite**: Order exists, state ∈ {`CONFIRMED`, `PACKED`}
- **PRD workflows**: `order-cancel`, `inventory-release`, `payment-void-refund`

### F-04 — Fulfillment dispatch

- **Trigger**: Operator klik "Dispatch" di WMS console
- **Actor**: Fulfillment Operator
- **Outcome**: Order → `DISPATCHED`, shipping label generated, customer notified
- **Prerequisite**: Order state = `PACKED`
- **PRD workflows**: `fulfillment-dispatch`, `notification-dispatch`

### F-05 — Customer return request

- **Trigger**: Customer request return dalam 14 hari setelah delivery
- **Actor**: Customer, Fulfillment Operator (approve)
- **Outcome**: Return authorized, RMA issued, refund scheduled
- **Prerequisite**: Order state = `DELIVERED`, within return window
- **PRD workflows**: `order-return`, `payment-refund`
